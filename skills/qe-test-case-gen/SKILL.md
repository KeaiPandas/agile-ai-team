# 测试用例生成 Skill

根据需求和验收标准生成全面的测试用例。

## 工作流程

### 1. 分析需求

**从 User Story 提取：**
- 用户角色
- 功能场景
- 验收标准
- 边界条件

### 2. 测试设计方法

**等价类划分：**
- 有效等价类：合法输入
- 无效等价类：非法输入

**边界值分析：**
- 最小值、最小值-1
- 最大值、最大值+1
- 空值、null

**场景法：**
- 基本流程
- 备选流程
- 异常流程

**状态迁移：**
- 初始状态
- 状态转换条件
- 最终状态

### 3. 测试用例格式

```markdown
## TC-XXX: [用例标题]

**模块：** [功能模块]
**优先级：** P0 / P1 / P2 / P3
**类型：** 功能 / 性能 / 安全 / 兼容
**关联 Story：** US-XXX

### 前置条件
- [ ] 条件一
- [ ] 条件二

### 测试步骤

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 打开登录页面 | 显示登录表单 |
| 2 | 输入手机号 13800138000 | 手机号显示正确 |
| 3 | 输入密码 password123 | 密码显示为 **** |
| 4 | 点击登录按钮 | 登录成功，跳转首页 |

### 测试数据

| 字段 | 有效值 | 无效值 |
|------|--------|--------|
| 手机号 | 13800138000 | abc, 123, 空 |
| 密码 | password123 | 空, < 6位 |

### 预期结果
- 登录成功
- 返回 token
- 跳转到首页

### 实际结果
[测试时填写]

### 状态
- [ ] 待测试
- [ ] 通过
- [ ] 失败
- [ ] 阻塞
```

### 4. 测试用例矩阵

```markdown
## 测试用例矩阵 - 登录功能

| ID | 场景 | 手机号 | 密码 | 预期结果 | 优先级 |
|----|------|--------|------|----------|--------|
| TC-001 | 正常登录 | 有效 | 有效 | 成功 | P0 |
| TC-002 | 手机号为空 | 空 | 有效 | 失败提示 | P0 |
| TC-003 | 密码为空 | 有效 | 空 | 失败提示 | P0 |
| TC-004 | 手机号格式错误 | abc | 有效 | 失败提示 | P1 |
| TC-005 | 密码错误 | 有效 | 错误 | 失败提示 | P0 |
| TC-006 | 账号不存在 | 未注册 | 任意 | 失败提示 | P1 |
```

### 5. 覆盖率检查

**功能覆盖：**
- [ ] 正常场景
- [ ] 异常场景
- [ ] 边界场景
- [ ] 安全场景

**验收标准覆盖：**
- [ ] 每个验收标准都有对应用例

```
workspace-qe/test-cases/[模块名]/
```
