# 障碍清除 Skill

识别、记录和解决 Sprint 中的阻塞问题。

## 工作流程

### 1. 障碍识别

**常见障碍类型：**
| 类型 | 示例 | 解决方向 |
|------|------|----------|
| 技术障碍 | API 文档缺失 | 联系相关负责人 |
| 资源障碍 | 测试环境不可用 | 协调运维 |
| 依赖障碍 | 等待第三方接口 | 调整计划或 Mock |
| 决策障碍 | 需求不明确 | 联系 PO 确认 |
| 人员障碍 | 团队成员请假 | 重新分配任务 |

### 2. 障碍记录格式

```markdown
## 障碍追踪

### B-001: [障碍标题]

**报告人：** Dev-A
**报告时间：** YYYY-MM-DD HH:mm
**影响范围：** US-001
**严重程度：** 🔴 高 / 🟡 中 / 🟢 低
**状态：** 🆕 新建 / 🔄 处理中 / ✅ 已解决

**描述：**
[详细描述障碍内容]

**影响：**
- 影响任务：US-001
- 预计延误：X 天

**解决方案：**
- [ ] 步骤 1
- [ ] 步骤 2

**负责人：** [解决负责人]
**预计解决：** YYYY-MM-DD
**实际解决：** YYYY-MM-DD

**复盘：**
[解决后的经验总结]
```

### 3. 障碍看板

```markdown
## 障碍看板 - Sprint XX

| ID | 描述 | 严重程度 | 状态 | 负责人 |
|----|------|----------|------|--------|
| B-001 | 等待 API 文档 | 🟡 中 | 🔄 处理中 | PO |
| B-002 | 测试环境异常 | 🔴 高 | 🆕 新建 | 运维 |

### 统计
- 总计：2
- 高优先级：1
- 已解决：0
```

### 4. 升级机制

**障碍升级流程：**
```
Level 1: 团队内部解决 (≤ 1天)
    ↓ 无法解决
Level 2: Scrum Master 协调 (≤ 2天)
    ↓ 无法解决
Level 3: PO/管理层介入 (> 2天)
```

**升级通知模板：**
```markdown
## 障碍升级通知

**障碍 ID：** B-XXX
**升级原因：** 超过 X 天未解决
**需要支持：** [具体需要什么帮助]

请相关方协助处理。
```

## 输出位置

```
workspace-scrum/blockers/active.md
workspace-scrum/blockers/archived/B-XXX.md
```

## 注意事项

- 记录要具体、可执行
- 高优先级障碍优先处理
- 解决后记录复盘经验
- 定期回顾障碍模式，预防重复
