# Dev Agent - 代码生成技能

## 概述

本技能指导 Dev Agent 根据产品需求、技术规格或用户指令，生成高质量的生产级代码。Dev Agent 应具备深入理解需求、选择合适技术方案、编写清晰可维护代码的能力。代码生成不仅仅是"写代码"，更是将需求准确转化为技术实现的过程。

## 触发条件

- 收到 User Story 或技术任务，需要编写新代码
- 收到"写代码"、"实现功能"、"开发"、"编码"、"编程"等指令
- 收到明确的功能需求描述，需要转化为代码实现
- 需要创建新的模块、组件、API 端点或数据库模型
- 收到来自 Sprint Backlog 的开发任务

## 流程

### 1. 需求理解

- 仔细阅读 User Story 或需求描述
- 识别核心功能点、边界条件和业务规则
- 明确输入/输出、数据结构和接口要求
- 如有疑问，向 PO 或团队成员澄清需求

### 2. 技术方案设计

- 分析现有代码库，了解项目架构和编码规范
- 选择合适的设计模式和架构方案
- 确定需要创建/修改的文件列表
- 评估技术可行性和潜在风险
- 考虑性能、安全性和可扩展性

### 3. 代码编写

- 遵循项目的编码规范和风格指南
- 编写清晰、有意义的变量名和函数名
- 添加必要的代码注释（解释"为什么"，而非"是什么"）
- 实现核心业务逻辑
- 处理边界情况和错误场景
- 确保代码模块化，职责单一

### 4. 代码自检

- 检查代码是否满足所有验收标准
- 验证边界条件和异常处理
- 确认没有硬编码的配置或密钥
- 检查代码可读性和可维护性
- 验证与现有模块的兼容性

### 5. 提交产出

- 将生成的代码提交到对应的功能分支
- 附上清晰的 commit message
- 通知团队成员代码已就绪

## 输入

- **User Story**: 包含验收标准的用户故事
- **技术规格**: 详细的接口设计、数据模型或架构文档
- **代码库上下文**: 现有项目的代码结构和编码规范
- **约束条件**: 性能要求、安全要求、兼容性要求等

## 输出

- **可运行的代码**: 符合需求的功能代码
- **代码文件**: 创建或修改的源代码文件
- **commit**: 包含清晰描述的 Git commit
- **实现说明**: 对关键设计决策的简要说明（如有必要）

## 注意事项

- **不要盲目生成代码**：先理解需求，再动手编写。不明确的地方务必确认。
- **遵循项目规范**：每个项目都有自己的编码风格，务必与现有代码保持一致。
- **考虑测试友好性**：编写的代码应当易于编写单元测试，避免过度依赖外部状态。
- **避免过度工程**：用最简单的方案解决问题，不要为了"优雅"而增加不必要的复杂度。
- **安全第一**：处理用户输入时做好校验和转义，避免 SQL 注入、XSS 等安全漏洞。
- **增量开发**：如果需求复杂，建议分步实现，每步都可验证。
- **保持沟通**：如果技术方案有重大决策，提前与团队沟通确认。
