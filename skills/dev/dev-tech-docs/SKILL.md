# 技术文档

## 概述

Dev Agent 根据代码和 Task 内容自动生成和维护技术文档，包括 API 文档、架构说明、开发指南等。文档与代码保持同步，确保团队成员（包括其他 Agent）能快速理解系统。

**在流程中的位置：** 代码生成 → **技术文档**（可并行） → 里程碑交付

## 触发条件

- Task 包含文档相关的验收标准
- 新增或修改 API 接口
- 架构发生变化
- 里程碑完成时需要更新项目文档

## 输入

- 项目代码（特别是接口定义、类型定义）
- `backlog/tasks/T-XXX.md`（Task 描述）
- 现有文档（了解格式和风格）
- 项目配置（技术栈信息）

## 流程

1. **识别文档需求**：判断当前变更是否涉及文档更新
   - 新增 API → 更新 API 文档
   - 架构变更 → 更新架构文档
   - 新增依赖/工具 → 更新开发指南
2. **分析代码**：提取关键信息
   - API 端点、参数、返回值、错误码
   - 模块职责和依赖关系
   - 配置项和环境变量
3. **生成文档内容**：
   - 保持与现有文档一致的格式和风格
   - 包含必要的代码示例
   - 说明使用场景和注意事项
4. **更新文档文件**：
   - API 文档 → `docs/api.md` 或接口注释
   - 架构文档 → `docs/architecture.md`
   - 开发指南 → `docs/guide.md`
5. **验证文档准确性**：确保文档与实际代码一致
6. **更新 Task 状态**：如文档是 Task 的验收标准之一，记录完成

## 输出

- 更新后的文档文件
- Task 工作记录更新

## 文件规范

### API 文档格式

路径：`docs/api.md`

```markdown
# API 文档

## 基础信息
- **Base URL:** `http://localhost:3000/api`
- **认证方式:** Bearer Token

## 接口列表

### POST /auth/login
用户登录。

**请求参数:**
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| email | string | 是 | 用户邮箱 |
| password | string | 是 | 密码 |

**响应:**
```json
{
  "token": "xxx",
  "user": { "id": 1, "name": "xxx" }
}
```

**错误码:**
| 状态码 | 说明 |
|--------|------|
| 401 | 邮箱或密码错误 |
| 429 | 请求过于频繁 |
```

## 注意事项

- **文档与代码同步**：代码变更后必须同步更新文档
- **注重实用性**：文档面向使用者和开发者，提供真正有用的信息
- **避免过度文档化**：代码本身足够清晰的地方不需要冗余文档
- **API 文档是硬要求**：所有对外暴露的接口必须有文档
- **保持简洁**：Agent 生成的文档应简洁明了，避免长篇大论
