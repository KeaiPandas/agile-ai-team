# Bug 报告 Skill

清晰记录缺陷，方便开发定位和修复。

## 工作流程

### 1. Bug 报告格式

```markdown
## BUG-XXX: [Bug 标题]

**严重程度：** 🔴 致命 / 🟠 严重 / 🟡 一般 / 🟢 轻微
**优先级：** 🔥 紧急 / ⚡ 高 / 📌 中 / 📋 低
**模块：** [功能模块]
**发现版本：** vX.X.X
**关联 Story：** US-XXX
**报告人：** QE Agent
**报告时间：** YYYY-MM-DD HH:mm

### 问题描述
[一句话描述问题]

### 复现步骤

1. 打开登录页面
2. 输入手机号 13800138000
3. 输入密码 password123
4. 点击登录按钮

### 预期结果
登录成功，跳转到首页

### 实际结果
页面显示"系统错误"，无法登录

### 复现概率
- [ ] 100%
- [ ] 偶发（X/10次）

### 环境信息
- 操作系统：Windows 11
- 浏览器：Chrome 120.0
- 设备：Desktop

### 附件
- 截图：![截图](./screenshots/BUG-XXX-1.png)
- 日志：[日志内容或链接]
- 视频：[录屏链接]

### 影响范围
- 影响用户：所有用户
- 影响功能：无法登录

### 根因分析（可选）
[开发填写]

### 修复方案（可选）
[开发填写]
```

### 2. 严重程度判定

| 级别 | 定义 | 示例 | 响应时间 |
|------|------|------|----------|
| 🔴 致命 | 系统崩溃、数据丢失 | 数据库崩溃 | 立即 |
| 🟠 严重 | 核心功能不可用 | 无法登录、无法支付 | 4小时 |
| 🟡 一般 | 功能异常但有变通 | 表单验证错误 | 24小时 |
| 🟢 轻微 | UI 问题、体验问题 | 文案错误、样式错位 | 下版本 |

### 3. 优先级判定

| 级别 | 定义 | 处理时机 |
|------|------|----------|
| 🔥 紧急 | 阻塞发布或影响核心业务 | 立即处理 |
| ⚡ 高 | 影响重要功能 | 本 Sprint |
| 📌 中 | 需要修复但不紧急 | 下 Sprint |
| 📋 低 | 可以延后 | 有空时处理 |

### 4. 通知开发者

**提交 Bug 后通知：**
```json
sessions_send({
  "sessionKey": "agent:dev:main",
  "message": "发现 Bug BUG-XXX：登录页面系统错误\n\n严重程度：严重\n优先级：高\n\n请尽快处理。\n\n详细报告：workspace-qe/bug-reports/BUG-XXX.md"
})
```

### 5. Bug 状态流转

```
🆕 新建 → 📋 已确认 → 🔄 处理中 → ✅ 已修复 → 🧪 待验证 → ✅ 已关闭
                    ↓                ↓
                ❌ 无法复现      ❌ 重新打开
```
