# QE Agent - 自动化测试技能

## 概述

本技能指导 QE Agent 编写、维护和执行自动化测试脚本。自动化测试是保障软件质量的关键手段，能显著提高测试效率、确保回归覆盖、支持持续交付。QE Agent 应根据项目特点选择合适的测试框架和策略，建立可持续维护的自动化测试体系。

## 触发条件

- 需要编写自动化测试脚本
- 需要执行自动化测试套件
- 收到"自动化测试"、"e2e"、"测试脚本"、"unit test"、"集成测试"等指令
- 新功能开发完成需要补充自动化测试
- CI/CD 流程需要集成自动化测试
- 回归测试需要自动化支持

## 流程

### 1. 测试策略制定

根据项目阶段和需求确定测试层次：
- **单元测试**: 测试单个函数/组件（快速、隔离）
- **集成测试**: 测试模块间交互（中等速度、部分隔离）
- **端到端测试 (E2E)**: 测试完整用户流程（较慢、真实环境）

### 2. 测试框架选择

根据技术栈选择合适的测试工具：
- 前端：Jest, Vitest, Cypress, Playwright
- 后端：Jest, Mocha, pytest, JUnit
- API：Supertest, Postman/Newman, RestAssured
- E2E：Cypress, Playwright, Selenium

### 3. 编写测试脚本

- **单元测试**: 遵循 AAA 模式（Arrange-Act-Assert）
  - 覆盖正常路径和异常路径
  - 使用 mock/stub 隔离外部依赖
  - 测试命名清晰描述预期行为

- **集成测试**:
  - 测试模块间的数据流和交互
  - 使用测试数据库或容器化的测试环境
  - 验证 API 的请求/响应格式

- **E2E 测试**:
  - 模拟真实用户操作流程
  - 覆盖关键业务场景（Happy Path + 异常路径）
  - 使用 Page Object Model 提高可维护性

### 4. 测试执行与报告

- 在 CI/CD 流水中集成测试执行
- 生成测试报告（通过率、覆盖率、失败详情）
- 对失败用例进行快速分析和修复
- 维护测试数据和环境

### 5. 测试维护

- 定期清理和更新过时的测试用例
- 随产品迭代补充新的测试用例
- 优化测试执行速度（并行化、选择性执行）
- 监控测试稳定性（减少 flaky test）

## 输入

- **被测代码**: 需要测试的功能或模块
- **需求规格**: 功能描述和验收标准
- **测试环境**: 测试服务器、测试数据、配置信息
- **现有测试**: 已有的测试用例和测试框架

## 输出

- **测试脚本**: 可执行的自动化测试代码
- **测试报告**: 执行结果、覆盖率、失败分析
- **CI 集成配置**: 测试在 CI 流水线中的配置

## 注意事项

- **测试金字塔原则**：大量单元测试 + 适量集成测试 + 少量 E2E 测试。
- **避免 flaky test**：不稳定的测试比没有测试更糟糕，会削弱团队对测试的信任。
- **测试独立可重复**：每个测试用例应独立运行，不依赖执行顺序和外部状态。
- **不要测试实现细节**：测试行为和结果，而非具体实现方式。
- **测试也是代码**：测试代码需要和产品代码一样保持高质量。
- **关注测试速度**：慢速的测试套件会拖慢开发节奏，优先保证快速测试的充分性。
