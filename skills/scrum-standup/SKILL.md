# 站会总结 Skill

跟踪 Sprint 进度，组织每日站会。

## 工作流程

### 1. 站会三问

每日站会核心问题：
1. **昨天完成了什么？**
2. **今天计划做什么？**
3. **有什么阻塞？**

### 2. 站会记录格式

```markdown
## 站会记录 - YYYY-MM-DD

### Sprint XX 进度
- **Sprint 目标：** [目标]
- **剩余天数：** X 天
- **完成故事点：** X / XX

### 成员汇报

#### Dev-A
- ✅ 昨天：完成登录页面 UI
- 📋 今天：实现登录 API 对接
- ❌ 阻塞：无

#### Dev-B
- ✅ 昨天：完成数据库设计
- 📋 今天：实现用户注册逻辑
- ❌ 阻塞：等待设计稿确认

### 阻塞项汇总
| 编号 | 描述 | 负责人 | 状态 |
|------|------|--------|------|
| B-001 | 等待设计稿确认 | PO | 处理中 |

### 今日重点
- [ ] 任务 1
- [ ] 任务 2
```

### 3. 进度追踪

**燃尽图数据：**
```markdown
## Sprint XX 燃尽图

| 日期 | 剩余故事点 | 理想剩余 |
|------|-----------|----------|
| Day 1 | 20 | 20 |
| Day 2 | 18 | 18 |
| Day 3 | 15 | 16 |
```

**进度判断：**
- 实际 ≈ 理想：进度正常
- 实际 > 理想：进度落后，需要关注
- 实际 < 理想：进度超前

### 4. 异常处理

**进度落后时：**
1. 分析原因（估时不准？阻塞？能力问题？）
2. 调整策略（加班？砍需求？支援？）
3. 通知 PO 可能的风险

## 输出位置

```
workspace-scrum/sprints/sprint-XX-standup-YYYY-MM-DD.md
```

## 注意事项

- 站会时间控制在 15 分钟内
- 关注阻塞，会后跟进
- 记录但不纠结细节问题
