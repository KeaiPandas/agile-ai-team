# Release Agent - CI/CD 触发技能

## 概述

本技能指导 Release Agent 管理项目的持续集成（CI）和持续部署（CD）流程。CI/CD 是现代软件开发的基础设施，它自动化了构建、测试、部署的全过程，让代码从提交到上线的路径更短、更安全、更可重复。Release Agent 应确保流水线稳定可靠，发布过程可追溯、可回滚。

## 触发条件

- 需要触发构建或部署流程
- 需要将代码部署到特定环境
- 收到"CI/CD"、"构建"、"部署"、"deploy"、"发布"等指令
- 新版本需要发布到 staging 或 production
- CI 流水线失败需要排查和处理
- 需要配置或修改 CI/CD 流水线

## 流程

### 1. 构建触发（CI）

- 代码提交/PR 合并后自动触发构建
- 确认构建配置正确（依赖版本、环境变量、构建参数）
- 监控构建过程和结果
- 构建失败时及时告警并通知相关开发者

### 2. 自动化测试集成

- 构建完成后自动运行测试套件
- 单元测试 → 集成测试 → E2E 测试（按顺序执行）
- 测试结果生成报告并归档
- 测试失败阻断部署流程

### 3. 环境部署（CD）

按环境分阶段部署：

- **开发环境 (Dev)**: 每次合并自动部署
- **测试环境 (Staging)**: 功能完成后手动/自动部署，用于 QA 验证
- **预发布环境 (Pre-prod)**: 发布前的最后验证
- **生产环境 (Production)**: 经过审批后的正式发布

### 4. 发布管理

- 确认发布前检查清单全部通过
- 执行发布前备份和数据迁移
- 按发布窗口执行生产部署
- 部署后进行冒烟测试验证
- 如有问题，执行回滚操作

### 5. 部署监控

- 监控部署后的系统健康指标
- 关注错误率、响应时间、资源使用等关键指标
- 如发现异常，及时触发告警和回滚

## 输入

- **代码变更**: 需要构建部署的代码版本
- **环境配置**: 各环境的配置文件和变量
- **部署策略**: 蓝绿部署、金丝雀发布等策略
- **审批信息**: 生产部署所需的审批记录

## 输出

- **构建产物**: 编译打包后的可部署产物
- **部署状态**: 各环境的部署成功/失败状态
- **部署报告**: 部署过程日志和结果
- **发布通知**: 部署完成后的团队通知

## 注意事项

- **生产部署要谨慎**：始终有回滚方案，永远不要在没有回滚能力的情况下发布。
- **环境变量不要混用**：开发、测试、生产环境的配置必须隔离，使用不同的环境变量。
- **敏感信息加密存储**：API Key、数据库密码等使用加密存储，不要明文写在配置中。
- **小步发布**：频繁小发布优于偶尔大发布，出问题影响范围小、定位快。
- **发布窗口**：选择低流量时段进行生产发布，降低风险影响。
- **部署可重复**：同一份构建产物部署到不同环境，避免"在我电脑上能跑"的问题。
