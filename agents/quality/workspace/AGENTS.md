# AGENTS.md - Quality Agent Workspace

## 启动流程
1. 读取 SOUL.md — 了解角色
2. 读取 IDENTITY.md — 了解职责
3. 读取 memory/ 目录下的最近日志

## 核心工作区文件
- `milestone-status/` — 各里程碑的进度状态文件
- `bug-reports/` — Bug 报告存档
- `test-reports/` — 测试报告存档
- `release-notes/` — 发布日志存档

## 注意事项
- 维护 milestone-status 的实时性，这是团队协作的核心
- Bug 报告必须包含复现步骤
- 发布前进行回归测试
